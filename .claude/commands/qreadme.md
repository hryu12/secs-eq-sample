현재 프로젝트를 분석해 아래 형식에 맞는 한글 README.md를 작성한다.

절차:
1. CLAUDE.md, 소스 파일, 설정 파일 등을 읽어 프로젝트를 파악한다.
2. 아래 형식으로 README.md 초안을 작성해 사용자에게 보여주고 승인을 받는다.
3. 승인 후 README.md를 저장한다.

---

## README 형식

```markdown
# <프로젝트명>

<한 줄 설명 — 무엇을 하는 프로그램인지>

## 개발 환경

- <언어 및 버전>
- <패키지 관리자>
- <주요 의존성>

## 아키텍처

```
<시스템 구성도 — ASCII 다이어그램으로 컴포넌트 간 관계 표현>
```

## 주요 기능

- <기능 1>
- <기능 2>
- ...

## 시작하기

```<shell>
# <설명>
<설치 명령>

# <설명>
<실행 명령>
```

## 설정

<설정 방법 한 줄 안내>

| 항목 | 기본값 | 설명 |
|------|--------|------|
| <항목> | `<기본값>` | <설명> |
```

---

규칙:
- 모든 내용은 한글로 작성한다.
- 설정 섹션이 없는 프로젝트는 해당 섹션을 생략한다.
- 아키텍처 다이어그램이 의미 없는 경우(단순 CLI 등) 생략한다.
- 코드 블록의 언어 태그는 실제 사용 언어에 맞게 지정한다 (powershell / bash / python 등).
